Overview

Introduction

Iothing is a node.js framework open source for easy IoT developements. It is designed to be used on boards like the Raspbery or the Intel Galileo. It provide some usefull features like an event manager, a cron manager and a GPIOs standard interface. It also provide an API to pilote and monitor your connected object.

Philosophy

Let's assume that our board with the plugged GPIOs is a thing. Iothing is created to add some fonctionality to it, that's why the framework is based on the decorator design pattern. We also want to create some connected objects without writting to much code, we prefer generate the application code from configurations files.

Configuration

To configure your application you must create a folder named config at the root of your project.

Thing configuration

Comming soon!

In this file we will be able to declare our actions, describe crons and create listeners.

In the file config/thing.js

module.exports = {
    name: 'my object',
    ip: '127.0.0.1',
    gpios: [
        {name: 'light', pin :1}
    ]
}
  • name : the name of your thing
  • ip : the IP of your thing
  • gpios : the list of all your GPIOs

Simple example

// Loading the Framework
var iothing = require('iothing')
// Getting the thing instance
var thing = iothing.get('thingInstance')

// You can manage you gpio by overriding the value function
thing.gpio({"pin":1}).value = function(value) {
    if (typeof value === 'undefined') {
        return "light is off"
    }
    return "light is on"
}

// Then access your gpios
console.log(thing.gpio({"pin":1}).value())

// You can attach actions to your thing
thing.action('turn-on-garden-lights', function() {
    console.log(thing.gpio({"pin":1}).value(1))
})

// You can attach crons to your thing
thing.cron('nights', '00 * * * * *', function () {
    // the thing can emit events
    thing.emit('night')
})

// You can attach listeners to your thing
thing.on('night', function () {
    thing.run('turn-on-garden-lights')
})
